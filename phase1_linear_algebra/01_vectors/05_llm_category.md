# 5. LLM・圏論との接続

## 5.1 LLM との接続

| 概念 | LLM での役割 |
|---|---|
| ベクトル空間 $\mathbb{R}^d$ | トークンの **埋め込み空間**。GPT-3 は $d=12288$、Llama-2 7B は $d=4096$ |
| 線形結合 | Transformer の中で「複数のトークンの埋め込みを重み付き和で混ぜる」操作（注意機構の出力） |
| 基底 | 概念的には、埋め込み空間の各「方向」が意味の軸に対応しうる（**特徴方向**） |
| 次元 | 表現力の上限を決める。次元が大きいほど多くの概念を区別できるが、計算コストと「次元の呪い」のトレードオフ |
| $L^2$ ノルム | コサイン類似度の正規化、勾配クリッピング、重み正則化 |
| $L^1$ ノルム | スパース化、Lasso 正則化（パラメータの多くを 0 にする） |
| $L^\infty$ ノルム | リプシッツ性の評価、敵対的摂動の解析 |

**分布仮説（Distributional Hypothesis）.** 「同じ文脈で出現する単語は意味も近い」という言語学の仮説（Harris, 1954）。これを数学化したのが Word2Vec などの埋め込みで、「意味の近さ」を「ベクトルの方向の近さ（コサイン類似度）」として表現する。**なぜ意味が方向に乗るのか** は、訓練の結果として「意味的に近い単語が同じ部分空間に集まるよう最適化される」から。

**有名な現象.** $\mathrm{vec}(\text{king}) - \mathrm{vec}(\text{man}) + \mathrm{vec}(\text{woman}) \approx \mathrm{vec}(\text{queen})$。これは **線形結合** の操作が意味的な操作（性別の置換）に対応する例。線形性が言語の構造の一部を捉えている、という驚くべき経験的事実。

## 5.2 圏論との接続（伏線）

体 $K$ 上のベクトル空間と線形写像のなす **圏 $\mathbf{Vect}_K$** は、圏論の最も重要な例の一つ。

- **対象（object）**: $K$ 上のベクトル空間
- **射（morphism）**: 線形写像
- **合成**: 写像の合成
- **恒等射**: 恒等写像

この圏の構造は Phase 5 で深く扱うが、いまの段階で押さえるべきは「ベクトル空間それ自体ではなく、**ベクトル空間の間の写像（線形写像）の集まり** が本質的な情報を担う」という視点。これが次単元（[[../02_matrices|行列と線形写像]]）への橋渡しになる。
