# Phase 1-1: ベクトル

## 1. ベクトルとは

ベクトルとは「大きさと方向を持つ量」。数学的には **数を並べたリスト** として扱う。

```
v = [1, 2, 3]
```

これは3次元ベクトル。成分が3つ（要素が3つ）あるので **3次元** という。

一般に n 個の数を並べたものを **n次元ベクトル** と書く：

```
v = [v₁, v₂, ..., vₙ]
```

縦に書くこともある（列ベクトル）：

```
v = ⎡v₁⎤
    ⎢v₂⎥
    ⎣v₃⎦
```

---

## 2. ベクトルの演算

### 足し算・引き算

同じ次元のベクトル同士は、成分ごとに足す・引く。

```
[1, 2] + [3, 4] = [1+3, 2+4] = [4, 6]
[1, 2] - [3, 4] = [1-3, 2-4] = [-2, -2]
```

### スカラー倍

スカラー（ただの数）をかけると、全成分にかかる。

```
3 × [1, 2] = [3, 6]
```

---

## 3. ノルム（ベクトルの長さ）

ベクトルの「長さ」を **ノルム** という。記号は `‖v‖`。

2次元の例：`v = [3, 4]` のとき、

```
‖v‖ = √(3² + 4²) = √(9 + 16) = √25 = 5
```

一般に n 次元ベクトル `v = [v₁, v₂, ..., vₙ]` のノルムは：

```
‖v‖ = √(v₁² + v₂² + ... + vₙ²)
```

### 単位ベクトル

ノルムが 1 のベクトルを **単位ベクトル** という。
任意のベクトルを自分のノルムで割ると単位ベクトルになる（**正規化**）：

```
v / ‖v‖
```

---

## 4. LLM・圏論との接続

| 概念 | LLM での使われ方 |
|---|---|
| 高次元ベクトル | 単語・文章・トークンを数値の列として表現（**埋め込み表現 / Embedding**） |
| ノルム | ベクトル間の距離計算、正規化に使う |
| 単位ベクトル | コサイン類似度の計算で必要（後の単元で扱う） |

**具体例:** GPT などのモデルでは、各トークン（単語のかけら）を 768次元や 4096次元のベクトルとして扱う。「猫」「犬」「車」がそれぞれ異なるベクトルに対応する。意味が近い単語ほど、ベクトルも近い方向を向く。

---

## 5. 確認問題

**Q1.** `v = [1, 2, 3]`, `w = [4, 0, -1]` のとき、`v + w` を計算せよ。

**Q2.** `v = [6, 8]` のノルム `‖v‖` を計算せよ。

**Q3.** `v = [1, 1]` を正規化（単位ベクトル化）せよ。

---

<details>
<summary>解答</summary>

**A1.** `[1+4, 2+0, 3+(-1)] = [5, 2, 2]`

**A2.** `‖v‖ = √(6² + 8²) = √(36 + 64) = √100 = 10`

**A3.** `‖v‖ = √(1² + 1²) = √2`
なので、`v / ‖v‖ = [1/√2, 1/√2] ≈ [0.707, 0.707]`

</details>
